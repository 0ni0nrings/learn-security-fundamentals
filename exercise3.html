Exercise 3: CloudTrail

Expected duration: ~10 minutes.

Click on a task to get hints.

Task 1: Enable CloudTrail

  1. Visit https://console.aws.amazon.com/cloudtrail/home?region=us-east-1
  2. Create a Trail
  3. Set a Trail name and a S3 Bucket (don't use the previous bucket name and remember that bucket names must be globally unique). Stay with the defaults for the other fields (do not enable any data events!).

Task 2: Create dummy API calls

  1. Visit https://console.aws.amazon.com/ec2/v2/home?region=us-east-1
  2. Visit https://console.aws.amazon.com/iam/home
  3. Visit https://console.aws.amazon.com/rds/home?region=us-east-1
  4. Visit https://console.aws.amazon.com/s3/home?region=us-east-1

Task 3: Look at the results

  1. Visit https://console.aws.amazon.com/cloudtrail/home?region=us-east-1#/events
  2. It may take a few minutes until you see the first events! There is a refresh button at the top right.

  4. Explore the S3 bucket. Navigate the "folder structure" and download one of the files that also contain all API activity.

Bonus 1: What are S3 Data Events?

For some services (only S3 at the moment) you can also get API calls for data related events: e.g. file was download. Depending on your access patterns this could generate many events! Have a look at the pricing before you activate this on a public S3 bucket!http://docs.aws.amazon.com/awscloudtrail/latest/userguide/logging-management-and-data-events-with-cloudtrail.html#logging-data-events
